# Локальная маркетинговая система

Marketing OS принципиально локальна.

Это не косметическая деталь, а часть позиции.

**Своё железо. Весь цикл.**

Система нужна не для того, чтобы добавить ещё одну AI-кнопку. Она нужна, чтобы маркетинг можно было вести как управляемый процесс: от позиции и идеи до текста, визуала, публикации, архива и повторного использования.

## Почему локальность важна

Маркетинговые данные — это не просто контент.

Это:

- брифы;
- гипотезы;
- клиентские сегменты;
- конкурентные формулировки;
- офферы;
- транскрипты;
- голосовые;
- черновики;
- визуальные шаблоны;
- публикации;
- история решений;
- данные клиентов и проектов.

Если всё это лежит в разных SaaS, система не видит себя.

Один сервис знает тексты. Другой — визуалы. Третий — публикации. Четвёртый — SEO. Пятый — автоматизацию. Но никто не видит весь маркетинговый цикл.

Marketing OS собирает этот цикл в одной локальной среде.

## Технический принцип

Marketing OS может использовать стандартный стек:

```text
Directus
PostgreSQL
Redis
Node.js
pm2
Markdown-vault
агенты
скиллы
шаблоны
воркеры
```

Сложность не должна быть видна пользователю как хаос.

Пользователь должен видеть рабочий экран: проекты, задачи, статусы, артефакты, публикации и ошибки.

## Что дает Directus

Directus становится панелью управления маркетинговым процессом.

Внутри можно держать:

- проекты;
- позиционирование;
- сегменты ЦА;
- стейкхолдеров;
- продуктовые офферы;
- транскрипты;
- SEO-статьи;
- карусели;
- рилсы;
- баннеры;
- контент-календарь;
- ссылки и UTM;
- статусы согласования;
- публикации;
- результаты воркеров.

Directus здесь не просто CMS.

Это операционный интерфейс маркетинга.

## Что дают агенты и скиллы

Агент — это роль и инструкция.

Скилл — это исполняемый модуль.

Так система остаётся расширяемой. Под новую нишу можно добавить нового агента. Под новый формат — новый шаблон. Под новый процесс — новый воркер.

Вместо закрытого «магического» сервиса появляется открытая архитектура:

```text
агент = markdown
скилл = mjs
шаблон = html / css / js
артефакт = markdown / файл / запись в Directus
```

## Что дает vault

Каждый важный артефакт должен сохраняться не только в базе, но и в Markdown-vault.

Это делает маркетинг не потоком одноразовых публикаций, а накоплением активов.

```text
vault/
  projects/
    project-name/
      positioning/
      transcripts/
      seo-articles/
      posts/
      carousels/
      banners/
      briefs/
      proof-points/
```

Vault можно читать, переносить, индексировать, подключать к Obsidian, использовать для Notepub и повторного использования материалов.

## Human-review pipeline

Marketing OS не должна обещать «полную автогенерацию без человека».

Это слабый и опасный фрейм.

Сильная позиция другая:

> AI готовит, система ведёт по маршруту, человек утверждает.

Базовый workflow:

```text
draft → in_review → approved → published
```

Так сохраняется скорость, но не теряется контроль.

## Чем это отличается от облачного SaaS

Облачный SaaS обычно продаёт функцию:

- написать текст;
- запланировать пост;
- собрать баннер;
- автоматизировать цепочку;
- вести календарь.

Marketing OS продаёт контроль над полным циклом.

```text
стратегия → контент → визуал → ревью → публикация → архив
```

Это не «ещё одна вкладка».

Это локальный маркетинговый штаб.

## Кому особенно важно

Локальность особенно важна, если:

- вы работаете с клиентскими данными;
- у вас есть закрытые брифы и гипотезы;
- вы не хотите хранить стратегию бренда в чужих SaaS;
- вы ведёте несколько проектов;
- вам нужна история артефактов;
- вы хотите расширять систему под себя;
- вы работаете на рынок РФ/СНГ и не хотите зависеть от западных инструментов.

## Позиционирование

Обычные сервисы говорят:

> «Мы поможем вам делать контент быстрее».

Marketing OS говорит иначе:

> **Вы сможете вести весь маркетинговый цикл на своём железе.**

Это не про скорость ради скорости.

Это про владение системой.
