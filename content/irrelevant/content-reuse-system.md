# Система повторного использования контента

Сильный контент редко должен жить в одном формате.

Один хороший разговор может стать статьей. Статья может стать серией постов. Посты могут вернуться в сайт как FAQ. FAQ может подсказать новую страницу. Так контент перестает быть расходником и становится накопительным активом.

## Проблема

Часто эксперт публикует материал один раз и теряет его.

Происходит так:

```text
идея → пост → лента → исчезновение
```

Через неделю нужно снова придумывать тему, снова формулировать тезисы, снова собирать контекст.

Это не система. Это ручной выпуск отдельных артефактов.

## Как мыслить иначе

Контент нужно рассматривать как материал, который может менять форму.

```text
разговор → статья → посты → карусель → FAQ → страница сайта → архив
```

Важна не только публикация, но и обратный ход: опубликованный материал должен возвращаться в базу знания, чтобы его можно было найти, связать и переиспользовать.

## Какие форматы можно получать

Из одного исходника можно собрать:

- SEO-статью;
- короткий пост;
- серию постов;
- карусель;
- тезисы для рассылки;
- FAQ;
- фрагмент оффера;
- блок для хаба;
- внутренние ссылки;
- идеи для следующих страниц.

## Ошибка: делать больше ради большего

Повторное использование не означает механическое размножение текста.

Смысл не в том, чтобы из одной идеи сделать двадцать слабых публикаций. Смысл в том, чтобы у сильной идеи появились подходящие формы под разные контексты.

## Связь с сайтом из заметок

Если контент хранится в Markdown-vault, он может возвращаться в сайт через [[notepub-hub|Notepub]].

Тогда публикации не исчезают в ленте, а накапливаются в структуре сайта.
