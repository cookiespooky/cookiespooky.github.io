# Marketing OS — локальная AI-маркетинговая система полного цикла

**Своё железо. Весь цикл.**

Marketing OS — это локальная AI-маркетинговая система для маркетолога, эксперта или небольшой команды, которым нужен не очередной чат с искусственным интеллектом, а управляемый цикл производства маркетинга.

Она соединяет стратегию, тексты, визуалы, публикации, статусы, human-review и архив артефактов в одной системе, которая работает на вашем железе и не превращает маркетинговые данные в россыпь файлов по чужим облакам.

```text
позиция → идея → текст → визуал → ревью → публикация → vault → повторное использование
```

## В чем проблема

Большинство команд уже используют AI, но используют его как умный чат.

Каждый раз приходится заново объяснять, кто вы, что продаёте, кому продаёте, какой у вас тон, какие офферы, какие ограничения и что уже было сделано.

Параллельно появляются другие инструменты: Canva для визуалов, планировщик для постов, таблица для контент-плана, отдельный сервис для SEO, отдельный чат для текстов, отдельная папка для файлов.

В результате маркетинг вроде бы автоматизирован, но на деле остается ручной склейкой.

Типичные симптомы:

- каждый AI-чат начинается почти с нуля;
- стратегия не связана с контентом;
- визуалы делаются отдельно от текста;
- SEO, посты, баннеры и рилсы живут в разных местах;
- данные клиентов, брифы и гипотезы расходятся по облачным сервисам;
- нет единого статуса: что в черновике, что на ревью, что опубликовано;
- успешные материалы нельзя нормально переиспользовать;
- команда платит за несколько SaaS, но всё равно копирует руками.

Marketing OS решает не задачу «сгенерировать ещё один текст».

Она решает задачу контроля над маркетинговым циклом.

## Позиция

**Marketing OS — это локальная операционная система маркетинга: стратегия по Трауту, тексты, визуалы и дистрибуция в одной открытой платформе, которая работает на своём железе и не отправляет данные в облако.**

Это не просто AI-копирайтер.  
Не просто автопостинг.  
Не просто no-code автоматизация.  
Не просто контент-планировщик.  
Не просто агентство в новом интерфейсе.

Это система, где маркетинговый артефакт проходит полный маршрут:

```text
стратегическая позиция
→ задача
→ агент / воркер
→ черновик
→ human-review
→ утверждение
→ публикация
→ Markdown-vault
→ повторное использование
```

## Против чего Marketing OS

Главный конкурент — не один конкретный сервис.

Главный конкурент — привычная связка:

```text
ChatGPT / Claude в браузере
+ Canva
+ Google Docs
+ таблицы
+ n8n / Make
+ SaaS-планировщик
+ ручная публикация
+ забытый архив
```

Эта связка кажется гибкой, но создает хаос: каждый инструмент видит только свой кусок, а маркетинговая память не накапливается.

Marketing OS — противоположный выбор.

Она собирает маркетинговый процесс вокруг проекта, позиции, артефактов, статусов и локального vault.

## Лестница категории

В голове клиента категория выглядит примерно так:

1. **ChatGPT / Claude в браузере** — первый рефлекс: «открою чат».
2. **Jasper / Copy.ai и другие облачные AI-копирайтеры** — тексты по подписке, данные в облаке.
3. **n8n / Make** — автоматизация потоков, но без маркетинговой стратегии.
4. **SMM-агентство или контент-фрилансер** — чужие руки, часто без системы внутри клиента.
5. **SaaS-планировщики контента** — календарь публикаций, но не полный цикл.
6. **Marketing OS** — полный цикл маркетинга на своём железе, с открытой архитектурой и human-review.

Marketing OS не пытается быть «ещё одним лучшим AI-инструментом».

Она занимает другую позицию: **локальная система полного цикла для тех, кто хочет владеть своим маркетинговым процессом.**

## Что внутри

Состав системы может развиваться, но ядро строится вокруг нескольких слоёв:

- **Directus** как панель управления проектами, контентом, статусами и артефактами;
- **локальный стек**: Linux / macOS / Windows, PostgreSQL, Redis, Node.js, pm2;
- **воркеры** для обработки задач, генерации, рендера и публикации;
- **агенты** как расширяемые markdown-инструкции под задачи маркетинга;
- **скиллы** как исполняемые модули для конкретных операций;
- **шаблоны** для визуалов, презентаций, баннеров, каруселей и рилсов;
- **Markdown-vault** для хранения истории проекта и артефактов;
- **human-review pipeline**, чтобы ничего не уходило в канал без подтверждения;
- **интеграции под рынок РФ/СНГ**: VK, РСЯ, Topvisor, WordPress и другие каналы.

## Почему локально

Маркетинговые данные — это не просто тексты.

Это брифы, инсайты, офферы, гипотезы, клиентские сегменты, конкурентные карты, промпты, черновики, визуальные шаблоны, публикации и история бренда.

Если всё это разложено по чужим сервисам, у команды нет настоящей маркетинговой памяти.

Локальная архитектура дает три преимущества:

1. **Контроль** — данные лежат у вас, а не в десятке облачных кабинетов.
2. **Повторяемость** — процесс строится через статусы и воркеры, а не через ручную склейку.
3. **Расширяемость** — агенты, скиллы и шаблоны можно дописывать под свою нишу.

## Почему это не агентство

Агентство делает маркетинг за вас.

Marketing OS делает так, чтобы у вас появилась собственная маркетинговая инфраструктура.

Это особенно важно для экспертов, инфобизнеса, локальных брендов и малых команд, где знание о продукте находится внутри, но не превращается в регулярные публикации, страницы, визуалы и рекламные материалы.

## Почему это не AI-генератор

AI-генератор создает отдельный результат.

Marketing OS управляет маршрутом результата.

```text
вход → задача → агент → черновик → ревью → публикация → архив
```

AI может помогать на каждом этапе, но система не строится на идее «пусть всё публикуется автоматически».

Наоборот: сильная часть Marketing OS — **human-review pipeline**. Машина готовит, человек утверждает.

## Ключевые сценарии

### Стратегия → контент

Проект начинается не с поста, а с карты позиции: категория, verbal nail, конкурент, лестница, доказательства, do-not-say, визуальный якорь.

Это защищает команду от типичной ошибки: производить много контента без ясного места в голове клиента.

### Транскрипт → серия материалов

Созвон, интервью, голосовая заметка или длинное объяснение превращаются в набор артефактов: статью, посты, карусель, FAQ, тезисы для видео и контент для сайта.

### SEO → статьи → сайт

SEO-план, кластеры и статьи могут попадать в Markdown-vault, а дальше использоваться для публикации через Notepub или WordPress.

### Идея → визуал

Из одной идеи можно собрать слайд, баннер, карточку, карусель, рилс или презентацию, если шаблоны и дизайн-токены живут внутри системы.

### Контент → архив → повторное использование

Опубликованный материал не исчезает в ленте.

Он становится частью vault: его можно найти, связать с проектом, обновить, превратить в другой формат или использовать как proof point.

## Для кого

Marketing OS подходит, если вы:

- уже используете Claude / ChatGPT, но устали от режима «каждый чат с нуля»;
- ведёте несколько проектов и хотите единую маркетинговую память;
- не хотите передавать брифы, клиентские данные и гипотезы в облачные SaaS;
- хотите регулярный выпуск SEO-статей, постов, баннеров, каруселей и рилсов;
- работаете на рынок РФ/СНГ и нуждаетесь в VK, РСЯ, Topvisor, WordPress;
- хотите open-source систему, которую можно форкнуть, расширять и внедрять клиентам;
- хотите строить маркетинг как процесс, а не как набор разрозненных генераций.

- «заменяет маркетолога»;
- «универсальный инструмент для всего»;
- «ещё один SaaS для контента».

Сильная формула проще:

> **Marketing OS — полный цикл маркетинга на своём железе.**
