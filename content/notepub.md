# Notepub — SEO-сайт из Obsidian и Markdown

Notepub — это публикационный слой, который превращает Markdown-заметки, Obsidian-графы, базы знаний и документацию в быстрый SEO-сайт.

Без тяжелой CMS.  
Без админки ради админки.  
Без переноса знания в чужую платформу.

## Для какой ситуации

Notepub нужен, когда контент уже есть, но он заперт внутри личного или командного архива.

Типичные ситуации:

- в Obsidian накопился экспертный граф;
- есть Markdown-документация, но нет нормального сайта;
- блог живет в заметках, но не работает на поиск;
- контент сложно переносить в CMS;
- хочется сохранить контроль над файлами;
- нужна быстрая публикация без vendor lock-in.

## Позиция Notepub

**Notepub — это SEO-сайт из Obsidian и Markdown без тяжелой CMS.**

Это не конструктор сайтов и не еще одна админка.  
Это способ опубликовать уже существующую структуру знания.

## Что делает Notepub

- читает Markdown;
- учитывает frontmatter;
- строит страницы по правилам;
- поддерживает типы контента;
- сохраняет внутренние связи;
- формирует SEO-структуру;
- генерирует быстрый HTML;
- оставляет контент у автора.

## Что видно на выходе

На выходе появляется не просто сайт, а публичная версия вашей базы знания:

```text
Markdown / Obsidian
→ структура страниц
→ перелинковка
→ SEO-метаданные
→ быстрый сайт
→ контент, который можно развивать дальше
```

## Почему это часть общей системы

Notepub закрывает важный разрыв: мысль должна выйти из личного графа в публичную структуру.

LAT помогает увидеть смысл.  
Self Tracker помогает увидеть граф.  
Notepub превращает граф в сайт.  
Marketing OS помогает постоянно пополнять этот сайт новым контентом.
