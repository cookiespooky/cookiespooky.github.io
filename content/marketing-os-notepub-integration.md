# Marketing OS + Notepub

Marketing OS и Notepub закрывают две разные части одной системы.

**Marketing OS** производит и управляет контентом.  
**Notepub** превращает Markdown-граф в SEO-сайт.

Вместе они дают полный цикл:

```text
идея → артефакт → markdown → граф → сайт → публикация
```

## Текущий статус

В Marketing OS уже есть база для интеграции:

- Markdown-артефакты в vault;
- проектная структура;
- Directus как редакторская панель;
- workflow согласования;
- контент-производство через воркеры.

Но пока нет встроенного пайплайна публикации SEO-сайтов через Notepub.

Это важная точка развития.

## Рекомендуемая архитектура

1. Использовать готовый шаблон Notepub-сайта.
2. Настроить режим `content_repo`.
3. Экспортировать утвержденный контент из Marketing OS в формат Notepub.
4. Перед публикацией запускать `notepub validate`.
5. После проверки запускать сборку и деплой.
6. Возвращать статус сборки обратно в Directus.

## UX-цель

Редактор не должен думать о GitHub Actions, ручной сборке и структуре репозитория.

Идеальный сценарий:

```text
утвердил контент → нажал «Обновить сайт» → получил ссылку на опубликованную страницу
```

## Почему это стратегически важно

Notepub делает знания публичными.

Marketing OS делает производство знаний регулярным.

Их связка превращает сайт эксперта из статичной витрины в систему накопления и публикации смыслов.

## Позиционирование связки

**Marketing OS + Notepub — это контентная инфраструктура для эксперта, который мыслит заметками, но должен быть видим рынком.**
