# Marketing OS + Notepub

Marketing OS и Notepub закрывают две части одной системы.

**Marketing OS** производит маркетинговые артефакты и ведёт их по статусам.  
**Notepub** превращает Markdown-граф в SEO-сайт.

Вместе они дают полный цикл:

```text
позиция → идея → артефакт → markdown → граф → сайт → публикация → обновление
```

## Зачем их связывать

У многих экспертов и команд контент появляется, но не становится публичным активом.

Посты уходят в ленту.  
Заметки остаются в Obsidian.  
Транскрипты лежат в папках.  
SEO-статьи живут отдельно от общей карты смыслов.  
Сайт обновляется рывками.

Связка Marketing OS + Notepub решает эту проблему.

Marketing OS делает производство регулярным.  
Notepub делает накопленное знание видимым рынку.

## Роль Marketing OS

Marketing OS отвечает за операционный слой:

- проекты;
- позиционирование;
- сегменты ЦА;
- задачи;
- транскрипты;
- SEO-планы;
- статьи;
- посты;
- визуалы;
- статусы;
- ревью;
- публикации;
- Markdown-vault.

Главный принцип:

```text
каждый артефакт имеет статус, владельца, следующий шаг и место в архиве
```

## Роль Notepub

Notepub отвечает за публичный слой:

- структуру сайта;
- Markdown-контент;
- внутренние ссылки;
- SEO-страницы;
- сборку;
- валидацию;
- публикацию.

Если Marketing OS — это производственный цех, то Notepub — это витрина и карта опубликованного знания.

## Рекомендуемая архитектура

```text
Directus
→ approved content
→ Markdown-vault
→ Notepub content repo
→ validate
→ build
→ deploy
→ status back to Directus
```

Редактор не должен думать о структуре репозитория и ручной сборке.

Идеальный сценарий:

```text
утвердил материал → нажал «обновить сайт» → получил ссылку на страницу
```

## Что публиковать через Notepub

Через Notepub логично публиковать:

- SEO-статьи;
- хабы;
- страницы офферов;
- гайды;
- методологические материалы;
- кейсы;
- библиотеки доказательств;
- карты тем;
- связанные страницы по проекту.

Не каждый пост должен попадать на сайт.

Но каждый сильный смысл должен иметь шанс стать частью публичного графа.

## Почему это стратегически важно

Marketing OS без Notepub может производить много артефактов.

Notepub без Marketing OS может остаться статичным сайтом.

Вместе они превращают маркетинг в систему накопления:

```text
производство → публикация → связывание → обновление → рост видимости
```

Это особенно важно для эксперта, который мыслит заметками, но должен быть видим рынком.

## Позиционирование связки

**Marketing OS + Notepub — это локальная система производства и публикации смыслов: Marketing OS ведёт артефакты по циклу, Notepub превращает утверждённый Markdown-граф в SEO-сайт.**

## Практический маршрут

1. Собрать карту проекта в Marketing OS.
2. Настроить статусы и типы артефактов.
3. Сохранять утверждённые материалы в Markdown-vault.
4. Определить, какие типы материалов попадают в Notepub.
5. Настроить экспорт в content repo.
6. Запускать validate перед публикацией.
7. Возвращать статус сборки в Directus.
8. Использовать сайт как публичный слой маркетинговой памяти.
